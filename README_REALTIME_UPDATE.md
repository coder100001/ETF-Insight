# 实时数据更新功能说明

## 功能概述

投资组合分析页面已添加手动触发实时数据更新的功能，用户可以随时点击按钮获取最新的ETF价格和组合分析数据。

## 功能特点

### 1. 一键更新
- 页面顶部新增"更新实时数据"按钮（绿色）
- 点击即可立即获取最新的ETF价格数据
- 无需刷新页面，数据即时更新

### 2. 加载状态提示
- 点击按钮后显示加载动画
- 顶部显示蓝色提示框："正在获取最新ETF价格..."
- 按钮变为不可用状态，防止重复点击

### 3. 成功反馈
- 更新完成后显示绿色成功提示
- 显示最后更新时间
- 所有数据卡片有轻微缩放动画效果
- 3秒后自动隐藏提示框

### 4. 错误处理
- 如果更新失败，显示红色错误提示
- 包含错误原因信息
- 按钮恢复可用状态
- 用户可以重试

### 5. 数据自动刷新
- 清除Redis缓存，确保获取最新数据
- 重新计算所有组合指标
- 更新页面所有显示数据：
  - 总投资
  - 当前价值
  - 资本利得
  - 税后年股息
  - 税前年股息
  - 股息税
  - 综合总收益
  - 综合收益率

## 使用方法

1. 访问投资组合分析页面：`http://127.0.0.1:8002/workflow/portfolio/`

2. 点击页面右上角的"更新实时数据"按钮

3. 等待几秒钟，系统会自动获取最新数据

4. 查看更新后的数据和分析结果

## 技术实现

### 前端
- **JavaScript函数**：`updateRealtimeData()`
- **异步请求**：使用Fetch API发送POST请求
- **状态管理**：显示/隐藏加载提示
- **数据更新**：`updatePageData()`函数更新DOM

### 后端
- **API端点**：`/workflow/api/update-realtime/`
- **视图类**：`UpdateRealtimeDataView`
- **功能**：
  - 清除缓存
  - 重新计算组合
  - 返回最新数据
  - 记录操作日志

### 数据流
```
用户点击按钮
    ↓
显示加载状态
    ↓
发送API请求
    ↓
清除Redis缓存
    ↓
获取最新ETF数据
    ↓
重新计算组合分析
    ↓
返回JSON响应
    ↓
更新页面显示
    ↓
显示成功提示
```

## 注意事项

1. **API限流**：频繁更新可能触发yfinance API限流，建议间隔至少10秒

2. **市场时间**：ETF数据在市场开市时更新更频繁

3. **网络延迟**：更新时间取决于网络速度和API响应时间

4. **缓存策略**：更新会清除所有缓存，确保数据最新

## 测试步骤

1. 启动Django服务器：
   ```bash
   python manage.py runserver 8002
   ```

2. 访问投资组合分析页面

3. 点击"更新实时数据"按钮

4. 观察加载状态和结果

5. 检查页面数据是否更新

## 故障排查

### 更新失败
- 检查网络连接
- 确认Django服务器运行正常
- 查看浏览器控制台错误信息

### 数据未更新
- 清除浏览器缓存
- 检查后端日志
- 确认Redis连接正常

### 按钮无响应
- 检查JavaScript控制台错误
- 确认CSRF token有效
- 查看API响应状态
